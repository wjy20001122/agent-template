# docs

这里存放项目级业务文档。原则是：**只放项目本身相关内容，不放 Agent 状态文件。**

## 文档清单

- `requirements.md`：需求说明。
- `architecture.md`：架构设计。
- `api.md`：接口文档。
- `deployment.md`：部署说明。
- `troubleshooting.md`：问题排查。

## 维护规则

- 需求变化写入 `requirements.md`。
- 架构变化写入 `architecture.md`。
- 接口变化写入 `api.md`。
- 部署变化写入 `deployment.md`。
- 排障经验写入 `troubleshooting.md`。
- Agent 状态、进度、决策、检查清单统一放 `.harness/`。
